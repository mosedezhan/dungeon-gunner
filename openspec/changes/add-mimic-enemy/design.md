## Context

游戏有六种敌人（Chaser / Rusher / Shooter / Giant / Bomber + 三精英变体），威胁类型覆盖持续接触、高速追击、远程弹幕、可读前摇 AOE、突袭爆炸。所有敌人的最优应对都是"尽快击杀"——缺少"追击决策"维度。

Mimic 是第一个不以"威胁玩家"为目标的敌人。它的价值在于**诱惑玩家冒险**：追它意味着暂时忽略场上其他敌人，承担掉血风险，但回报是高价值掉落。

**约束条件：**
- 无外部美术资源，所有视觉必须 BootScene 程序化生成
- 刷出在可见区域内——与所有其他敌人（屏幕外刷出）不同
- 不可破坏现有敌人行为
- 命名空间必须对齐 `src/entities/CLAUDE.md` 的四字段约定
- 新增 Enemy 子类时必须同步更新 `src/debug/registry.js`

## Goals / Non-Goals

**Goals:**
- Mimic 成为第一种"逃跑型"敌人，以"伪装 → 触发逃跑 → 限时击杀"为核心差异化机制
- 双计时器设计：Wandering 15s 超时 + Fleeing 10s 超时，创造"追不追"的实时决策
- 保底 XP + 25% 升级 / 75% 满充能——奖励值得冒险
- 棕金色调宝箱外观，与所有现有敌人（绿/蓝/红/橙）视觉区分
- W4 首现，给玩家 3 波基础学习后再引入决策维度

**Non-Goals:**
- 宝箱/钥匙系统——Mimic 掉落直接触发升级或充能，不引入新物品类型
- 迷你地图标记——玩家需要自己看到它
- 多种 Mimic 变体（精英版等）——留给后续 change
- 新增升级卡——升级池扩展走单独 change

## Decisions

### 1. 双阶段状态机（Wandering / Fleeing），独立计时器

**选择**：Mimic 持有 `state` 字段（`'wandering' | 'fleeing'`），两个独立计时器。

```
刷出 → Wandering（15s 超时）
         │
         └─ 玩家进入 triggerRange → Fleeing（10s 超时）
                                       │
                                       ├─ 被击杀 → die() + 掉落
                                       └─ 超时 → 消失（无掉落）

Wandering 15s 到期且未触发 → 直接消失（无掉落）
```

**理由**：
- 两个独立计时器让 Mimic 最长存活 15s（从未被触发）或 触发时刻+10s（取较早者）
- 15s 徘徊窗口让玩家有足够时间注意到它并做出决策
- 10s 逃跑窗口足够追击但不会无限拖延节奏

### 2. Wandering 行为：以刷出点为圆心，半径 30px 内随机游荡

**选择**：记录 `_spawnPoint`，每次随机选取 30px 半径内的目标点，以 20px/s 速度移动过去，到达后停顿 500~1000ms，再选下一个点。

**理由**：
- 极慢的游荡速度（20px/s）让它看起来像一个放在地上的宝箱在微微晃动
- 30px 半径保持它基本在原地，不会漂移太远
- 停顿间歇模拟"静止的宝箱"错觉，强化伪装感

### 3. Fleeing 行为：持续远离玩家 + 锯齿偏移 + 恐慌加速

**选择**：
- 主方向 = `(self.pos - player.pos).normalize()`
- 每隔 0.5s 随机叠加 ±30° 的偏移，产生锯齿路线
- 速度 = `baseSpeed(120) + panicAccel * (elapsed / fleeTime)²`，末期达到 160

**理由**：
- 纯直线逃跑太容易被预判和截击
- 锯齿偏移让追击需要预判走位，增加技巧性
- 恐慌加速制造时间压力：越晚追越难追上
- 用平方曲线让前期加速缓慢、末期急剧加速，给玩家"再不追就来不及了"的感觉

### 4. 刷出位置：玩家可见区域内，距玩家 250~400px

**选择**：在玩家周围随机方向、250~400px 距离处刷出。如果该位置在世界边界外，回弹到合法位置。

**理由**：
- 与所有其他敌人（屏幕外刷出）完全不同的刷出逻辑，确保玩家能看到
- 250px 最近距离避免刷在脚下（没有惊讶感），400px 最远保证在屏幕可见范围内
- 刷出时播放短暂金光闪烁效果，引导注意力

**替代方案**：
- 屏幕外刷出 + 走向玩家：rejected，玩家可能没注意到，浪费了"伪装宝箱"的设计

### 5. 掉落实现：在 handleEnemyDeath 中分支处理

**选择**：`GameScene.handleEnemyDeath(enemy)` 中检测 `enemy.texture.key.startsWith('mimic')`（或给 Enemy 加 `enemyType` 属性），如果是 Mimic：
1. 正常生成 XP 球（8 XP）
2. 25% 概率调用升级流程（同 WaveManager 的升级触发）
3. 75% 概率直接设置 `player.skillCharges = player.stats.skillChargesMax`

**理由**：
- 掉落逻辑集中在 GameScene，与现有模式一致
- 升级触发复用现有的 UpgradeScene，零额外 UI 成本
- 满充能直接修改 player 状态，简单可靠

### 6. 消失效果：渐隐 + 小粒子散开

**选择**：超时消失时，alpha 从 1 渐变到 0（300ms），同时释放 3~5 个小粒子（棕色碎片），然后 `destroy()`。

**理由**：
- 渐隐比瞬间消失更有"逃离"的感觉
- 粒子效果给玩家一个明确的视觉反馈："它跑了"
- 简单实现：tween alpha + 几个短寿命 sprite

### 7. 波次安排：W4+ 每波 15% 概率

**选择**：
```
W4+: maybeSpawnMimic() 在波次第 3~6s 随机时刻调用
     15% 概率刷出一只 Mimic
     每波最多 1 只
```

**理由**：
- W4 时玩家已熟悉 Chaser/Rusher/Shooter 基础应对
- 15% 概率 = 平均每 6~7 波遇到一只，稀有但不至于被遗忘
- 不随波次增加概率——Mimic 的价值在决策本身，不在数量

## Risks / Trade-offs

### Risk 1: Mimic 刷在可见区域但玩家忙于战斗没注意到

**Mitigation**: 刷出时的金光闪烁效果足够醒目。如果实测发现仍然被忽略，可考虑添加一个短暂的音效提示（但本项目暂无音效系统）。

### Risk 2: 追击 Mimic 时被其他敌人击杀，体验负面

**Mitigation**: 这是设计意图——追击本身就是高风险行为。Mimic 不造成伤害（contactDamage=0），风险完全来自玩家忽视的其他敌人。玩家需要学会在安全时追击。

### Risk 3: Mimic 逃跑出世界边界

**Mitigation**: 逃跑方向在应用前 clamp 到世界边界内。如果已到达边界，改为沿边界方向逃跑。WaveManager 使用 `world.getSpawnRing()` 管理世界范围，Mimic 逃跑时读取同一范围。

### Risk 4: 25% 升级触发打断游戏节奏

**Mitigation**: UpgradeScene 本身已设计为暂停 GameScene 的叠加层，波次间也会触发。Mimic 的升级触发与波次结束升级体验一致，不是新的节奏打断。

### Trade-off: Mimic 不参与正常 mix 刷怪池，独立 maybeSpawnMimic 逻辑

Mimic 的刷出逻辑与所有其他敌人不同（可见区域、概率触发、固定数量），不适合混入 `mixForWave()` 的随机抽取。独立的 `maybeSpawnMimic()` 方法虽然增加了 WaveManager 代码，但让刷出逻辑清晰可控，且不会意外增加 Mimic 数量。
