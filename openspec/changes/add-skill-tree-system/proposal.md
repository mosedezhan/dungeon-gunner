## Why

当前升级系统是扁平随机抽卡：每次升级从 `UPGRADES[]` 列表抽 3 张，部分卡有 `requires` 前置条件。法师有冰霜新星 / 法力虹吸 → 奥术风暴的雏形树，战士有 bt_duration / bt_slow → time_stop 的雏形树，但都只有 2 条 2 层的浅路径。

问题：
- 升级池太浅，20 张卡在 Lv15 后开始反复抽重复卡
- 没有路径感——"冰霜"和"虹吸"在玩家感知中只是两张独立升级卡，不是"冰霜之路"
- build 分化不足——同职业不同局的选择很快趋同
- 没有"铺印记 → 反应 → 连锁"的化学系统，跨路径投资没有协同奖励

灵感来源：绝区零「怪奇旅伴」小游戏的角色成长技能树 + Genshin 元素反应的跨路径融合。

## What Changes

### 总体架构：树形抽卡 + 独立面板 + 元素反应 / 姿态融合

**不替换抽卡体验**，而是把后台卡池从扁平列表升级为树结构——前置节点未解锁时后续节点不出现在卡池里。抽到的仍是 3 张随机卡，但树结构保证成长有方向。

新增独立 `SkillTreeScene`（TAB 键打开），展示完整技能树 + 反应图谱 / 姿态融合矩阵，让玩家看到全貌并规划路线。

两个职业用不同的融合逻辑（这是职业差异化的核心）：
- **法师：元素反应** — 四种元素印记（凝霜 / 带电 / 扭曲 / 灼烧），对已有印记 A 的敌人施加印记 B 触发反应，反应产物可连锁
- **战士：姿态叠加** — 四种战斗姿态（子时 / 嗜血 / 扎根 / 共振），同时处于两种姿态时下次挥砍自动变为融合攻击

### 法师四路径 + 终极

```
霜寒之路 — 控场 + 区域封锁
  T1 冰晶弹(×3)    T2 冰霜新星(×2)    T3 极寒领域 / 碎冰    T4 暴风雪 / 永冻

雷鸣之路 — 连锁天罚
  T1 链式闪电(×3)  T2 静电充能         T3 雷暴 / 过载(×2)    T4 避雷针 / 唤雷

虚空之路 — 撕裂现实
  T1 虚空弹(×3)    T2 空间折叠         T3 奇点 / 法力虹吸(×2) T4 裂隙行走 / 坍缩

余烬之路 — 烈焰不熄
  T1 余烬弹(×3)    T2 烈焰蔓延         T3 炼狱 / 爆破         T4 凤凰 / 灰烬风暴

共用根基：锋芒(×∞) / 敏捷连射(×5) / 共振核心(×3) / 疾行(×3)

终极：奥术风暴 (需两条路径各有一枚 T3+ 节点)
```

|---|---|---|---|---|

| 冻燃 | 凝霜 + 灼烧 | 2.0 | ☆ | 纯白核弹 |
| 雷虚裂隙 | 带电 + 扭曲 | 0.4 | ★★★★ | 空间裂口放电 |
| 雷焰引爆 | 带电 + 灼烧 | 0.5 | ★★★★★ | 火焰闪电增殖链 |
| 虚空焚烧 | 扭曲 + 灼烧 | 1.2 | ★★ | 等离子光束狙杀 |

### 战士四路径 + 终极

```
时空之路 — 时间操控
  T1 子弹时间+(×2)  T2 引力井(×2)    T3 时空裂隙 / 时空斩   T4 时停 / 倒转

血怒之路 — 伤口是燃料
  T1 血刃(×3)      T2 盛怒           T3 嗜血 / 处决(×2)     T4 狂暴 / 死亡契约

  T1 铁壁(×2)      T2 要塞           T3 扎根 / 战吼         T4 不破 / 最后一站

波动之路 — 震荡万物
  T1 震波斩        T2 共鸣           T3 潮力 / 音爆(×3)     T4 海啸 / 地裂

共用根基：锋芒(×∞) / 狂热(×5) / 共振核心(×3) / 疾行(×3)

终极：绝对时停 (需时空路径 T4)
```

### 战士六种姿态融合

| 融合 | 姿态组合 | 倍率 | 触发难度 | 定位 |
|---|---|---|---|---|
| 血色时间 | 子时 + 嗜血 | 2.5 | 10% | 凝固血刃 |
| 时空波动 | 子时 + 共振 | 3.0 | 17.5% | 360° 时停冲击环 |
| 血浪 | 嗜血 + 共振 | 2.0 | 28% | 血色能量波 |
| 不屈铁壁 | 嗜血 + 扎根 | 1.5 | 20% | 受击冲击波 |
| 要塞时间 | 子时 + 扎根 | — | 12.5% | 范围无敌 |
| 磐波 | 扎根 + 共振 | 3.5 | 35% | 地震裂纹波 |

### 连锁规则

```
衰减: damage_k = damage_0 × 0.7^k
最大链数: 4
同敌冷却: 1s
搜索半径: 120px × 0.85^k
反应触发: 命中带印记敌人 + 印记 ≠ 本反应消耗的两个印记 → 可连锁
```

### 全屏反馈分级

```
第2链: 微震 + 边缘晕影
第3链: 中震 + 晕影加深 + 0.1s 子弹时间
第4链: 强震 + 闪屏 + 0.2s 子弹时间 + 晕影脉冲
```

### SkillTreeScene (独立面板)

- TAB 键开关，暂停 GameScene
- 左侧 (780px)：技能树节点 + 连线
- 右侧 (500px)：法师=反应六角图谱 / 战士=姿态融合矩阵
- 底栏：等级 + 已点节点数 + 路径投入概览
- 悬停节点：效果详情 + 解锁预告（"选这个将解锁: X, Y, Z反应"）
- 悬停反应/融合：连锁预测 + 当前 build 实际伤害计算
- 升级时 UpgradeScene 底部增加"将解锁"提示

## Capabilities

### New Capabilities

- `skill-tree`: 树形升级结构 + 独立面板，含 prerequisites 链、路径分层、反应图谱 / 姿态矩阵展示
- `element-mark`: 敌人元素印记系统，含四种印记施加 / 持续 / 视觉 / 覆盖规则
- `element-reaction`: 六种元素反应 + 连锁引擎，含反应检测、伤害计算、连锁传播、全屏反馈分级
- `warrior-stance`: 四种战士姿态 + 六种融合，含姿态检测、融合触发、蓄力预告、残留效果
- `skill-tree-vfx`: 四元素视觉 DNA + 六反应融合态 + 连锁路径标记 + 全屏反馈的完整视觉系统

### Modified Capabilities

- 升级系统：从扁平列表升级为树结构（prerequisites 替代部分 requires）
- UpgradeScene：底部增加"将解锁"预告
- HUDScene：增加反应浮标 + 融合指示器

## Impact

### 受影响代码

几乎所有核心文件——这是游戏成长系统的地基重做。具体：
- `src/config.js`：大量新配置（TREE / REACTION / FUSION / SCREEN + ~50 个新 UPGRADES 条目）
- `src/entities/Enemy.js`：新增印记系统（markType / markUntil / mark tint）
- `src/entities/Player.js`：新增元素/姿态状态追踪
- `src/systems/Upgrades.js`：prerequisites 过滤逻辑
- `src/scenes/GameScene.js`：反应引擎 + 姿态检测 + 全屏反馈
- `src/scenes/UpgradeScene.js`：将解锁预告

### 新增文件

- `src/scenes/SkillTreeScene.js`
- `src/systems/ReactionEngine.js`（反应检测 + 连锁逻辑，独立模块）
- `src/systems/ElementVFX.js`（元素 / 反应 / 连锁的 VFX 工厂，独立模块）

### 破坏性变更

- `UPGRADES[]` 结构变更：现有升级卡需要补充 `prerequisites` / `tier` / `path` 字段
- `randomUpgrades()` 逻辑从扁平过滤改为树结构过滤
- Enemy 基类新增印记属性（向后兼容，默认无印记）

### 依赖

无外部依赖。纯 Phaser 3 内置 API。

## Phasing (实施拆分)

本 change 超出单次 ≤3 天限制，拆分为以下子 change 按序实施：

### Phase 1: 树结构 + 法师霜寒路径 (基础验证)
- config.js: TREE 配置框架 + UPGRADES 重构为树结构
- Upgrades.js: prerequisites 过滤
- 法师霜寒路径所有节点（T1-T4）实现
- 元素印记施加机制（凝霜 only）
- 无反应、无连锁、无 SkillTreeScene
- **Playtest 目标**: 法师走霜寒路径，感受树形抽卡的成长节奏

### Phase 2: 法师雷鸣路径 + 反应引擎 + 碎冰雷暴
- 雷鸣路径所有节点实现
- ReactionEngine.js: 反应检测 + 连锁引擎
- 碎冰雷暴反应（霜+雷）完整实现含连锁
- 全屏反馈分级
- **Playtest 目标**: 双路径 build，体验碎冰雷暴连锁清屏

### Phase 3: 法师虚空 + 余烬 + 全反应
- 虚空路径 + 余烬路径节点实现
- 剩余 5 种反应实现
- ElementVFX.js: 四元素视觉 DNA + 六反应融合态 VFX
- 连锁路径标记 VFX
- **Playtest 目标**: 全反应可用，六种反应各有辨识度

### Phase 4: 战士四路径 + 姿态融合
- 战士全部路径节点实现
- 姿态检测系统
- 六种融合实现
- 融合蓄力预告 + 残留效果 VFX
- **Playtest 目标**: 战士走双路径，体验姿态融合的"事件感"

### Phase 5: SkillTreeScene + UpgradeScene 增强
- SkillTreeScene 完整实现（树布局 + 反应图谱 / 姿态矩阵）
