# 经验与升级系统 GDD

## 1. 概述

定义经验值的累积、升级公式和卡牌选择流程。经验球和血球的掉落规则见掉落物系统 GDD。经验值累积到阈值后自动升级，触发卡牌选择界面。

## 2. 玩家幻想

击杀敌人后看到经验球飞散开来，走近或经过时自动吸收，经验条稳步填满。升级那一刻有明显的反馈（视觉+音效），然后从三张卡牌中选择一张强化，每次升级都让角色变得更强。

## 3. 详细规则

### 经验球掉落
经验球的具体掉落规则、拾取机制和性能优化详见掉落物系统 GDD。

### 升级公式

### 升级公式
```
xpToNextLevel = 10 + level × 5

示例：
Level 1 → 2: 10 + 1×5 = 15 XP
Level 2 → 3: 10 + 2×5 = 20 XP
Level 3 → 4: 10 + 3×5 = 25 XP
Level 5 → 6: 10 + 5×5 = 35 XP
Level 10 → 11: 10 + 10×5 = 60 XP
```

### 升级流程
1. 经验值达到 `xpToNextLevel` 时自动触发升级
2. 游戏暂停（敌人停止移动和攻击）
3. 显示卡牌选择界面（3 张随机卡牌）
4. 玩家选择一张后游戏继续
5. 玩家等级 +1，经验值溢出部分保留

### 等级效果
- 等级本身不直接增加属性
- 属性增长完全来自升级时选择的卡牌
- 等级仅作为经验需求的计算参数

### 经验球合并（可选优化）
见掉落物系统 GDD。

## 4. 公式

### 升级经验需求
```
xpToNextLevel = 10 + level × 5

变量定义：
- level: 当前等级（从 1 开始）
- 返回值: 升到下一级所需的总经验

范围：
- level ∈ [1, ∞)
- xpToNextLevel ∈ [15, ∞)
```

### 总经验累积（前 N 级）
```
totalXpForLevel(N) = Σ(k=1 to N-1) (10 + k×5)
                   = 10(N-1) + 5×(N-1)N/2

示例：
Level 5 需要 totalXp = 10×4 + 5×4×5/2 = 40+50 = 90 XP
Level 10 需要 totalXp = 10×9 + 5×9×10/2 = 90+225 = 315 XP
```

### 经验球散射
```
scatterOffset = randomUnitVector() × random(20, 40)
spawnPosition = enemyDeathPosition + scatterOffset
```

## 5. 边缘情况

- **同时拾取多个经验球触发升级**：正常升级，溢出经验保留到下一级
- **升级时玩家正在受击**：暂停状态下无敌帧暂停计时，选择卡牌后继续
- **满级（无限等级）**：无等级上限，公式支持无限延伸
- **经验球过多导致性能问题**：见掉落物系统 GDD 磁吸清理机制
- **升级期间新经验球掉落**：暂停期间敌人不死亡，不会产生新经验球

## 6. 依赖

### 上游依赖
- **敌人系统**：提供每种敌人的 xpReward 值
- **角色系统**：需要玩家的当前位置和等级

### 下游依赖
- **卡牌系统**：升级时触发卡牌选择
- **HUD**：显示经验条、等级数字

### 双向依赖
- **波次系统**：波次击杀产生经验，高波次更多经验球

## 7. 可调参数

| 参数 | 默认值 | 安全范围 | 影响方面 |
|------|--------|---------|---------|
| 基础经验需求 | 10 | 5-20 | 升级速度 |
| 每级递增 | 5 | 2-10 | 后期升级难度 |
| 经验球拾取半径 | 20px | 10-50 | 拾取便利性 |
| 经验球散射半径 | 20-40px | 10-80 | 掉落分散度 |
| 磁吸清理阈值 | 50 个 | 30-100 | 性能保护 |

## 8. 验收标准

- [ ] 敌人死亡后在死亡位置生成经验球，数值对应 xpReward
- [ ] 玩家靠近经验球 20px 内自动拾取，经验值增加
- [ ] 经验值达到阈值时自动升级，等级 +1
- [ ] 升级时游戏暂停，显示 3 张随机卡牌
- [ ] 选择卡牌后游戏继续
- [ ] 经验值溢出正确保留到下一级
- [ ] HUD 显示当前等级和经验进度条
- [ ] Level 1→2 需要 15 XP，Level 5→6 需要 35 XP
