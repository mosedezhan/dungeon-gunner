# 法师攻击系统 GDD

## 1. 概述

定义法师的远程子弹攻击机制。法师朝鼠标方向发射子弹，子弹直线飞行并命中第一个敌人。所有攻击参数数据驱动，卡牌可改变子弹形态（穿透、爆炸、分裂、多发），产生质变式的能力升级。

## 2. 玩家幻想

按住鼠标，子弹一颗接一颗朝光标方向飞出去。前期单发子弹伤害低但频率快，靠走位持续输出。拿到穿透弹后子弹穿过敌人不消失，一条直线打一串。拿到爆裂弹后每颗子弹命中都爆炸，AOE 清场。拿到弹幕风暴后一发三颗扇形子弹，配合爆炸和分裂，满屏弹幕。

## 3. 详细规则

### 基础攻击行为

- 按住鼠标左键时按 fireRateMs 间隔自动发射子弹
- 每次发射 bulletCount 颗子弹（默认 1）
- 子弹沿攻击方向直线飞行，速度为 bulletSpeed
- 子弹命中敌人后按命中处理链执行效果

### 子弹属性

| 属性 | 默认值 | 说明 |
|------|--------|------|
| damage | 10 | 每颗子弹伤害 |
| bulletSpeed | 260 px/s | 子弹飞行速度 |
| bulletCount | 1 | 每次发射子弹数 |
| spreadAngle | 0° | 多子弹扇形总角度 |
| piercing | 0 | 可穿透敌人数 |
| bulletBehavior | straight | 子弹飞行行为 |

### 子弹生命周期

1. 从角色中心发射，方向 = attackDirection
2. 每帧沿方向移动 bulletSpeed × deltaTime
3. 碰到敌人时触发命中处理
4. 子弹存活时间上限：2000ms
5. 超出屏幕 200px 后回收

### 多子弹发射（弹幕风暴）

- bulletCount > 1 时，子弹均匀分布在 spreadAngle 扇形内
- 扇形中心轴 = attackDirection
- 每颗子弹角度 = attackDirection + spreadAngle × (i / (bulletCount-1) - 0.5)

```
示例（弹幕风暴：3 颗，30°）：
- 子弹 0: direction - 15°
- 子弹 1: direction + 0°
- 子弹 2: direction + 15°
```

### 命中处理链（按顺序）

```
子弹命中敌人时：
1. 对敌人造成 damage 点伤害
2. 遍历 statusEffects 数组，独立判定施加状态
3. 如果 piercing > 0 → 子弹穿透继续飞行，piercing - 1
4. 如果 explodeOnHit → 在命中点产生爆炸
5. 如果 splitOnHit → 在命中点生成分裂子弹
6. 如果 piercing == 0 且未分裂 → 子弹销毁
7. 遍历 triggers.onHit，执行所有命中触发器
8. 如果敌人死亡：遍历 triggers.onKill，执行所有击杀触发器
```

### 穿透弹

- 子弹命中敌人后不销毁，继续飞行
- 穿透后子弹方向和速度不变
- 同一颗子弹对同一敌人只命中一次（避免穿透来回打同一个人）
- piercing 值 = 可穿透的敌人数（穿透弹卡牌设为 1）

### 爆裂弹

- 子弹命中敌人时在命中位置产生爆炸
- 爆炸为圆形范围，半径 explosionRadius（40px）
- 爆炸对范围内所有敌人造成 damage × explosionDamageRatio 伤害
- 爆炸伤害不触发状态施加（避免连锁灼烧过于强大）
- 爆炸有视觉特效（扩散圆环）

### 分裂弹

- 子弹命中敌人后在命中位置生成 splitCount 颗分裂子弹
- 分裂子弹均匀分布在 360° 方向
- 每颗分裂子弹伤害 = 原子弹 damage × splitDamageRatio
- 分裂子弹存活时间：1000ms
- 分裂子弹不继承穿透/爆炸/分裂能力（防止无限递归）

### 组合交互

| 组合 | 行为 |
|------|------|
| 穿透 + 爆裂 | 子弹每次穿透都爆炸，穿过 3 个敌人产生 3 个爆炸 |
| 穿透 + 分裂 | 子弹穿透后继续飞行，到达射程尽头或下次命中时才分裂 |
| 爆裂 + 分裂 | 先爆炸再分裂，分裂子弹不爆炸（除非持有弹幕地狱隐藏卡） |
| 弹幕风暴 + 穿透 | 3 颗子弹各自独立穿透 |
| 弹幕风暴 + 爆裂 | 3 颗子弹各自独立爆炸 |
| 弹幕风暴 + 分裂 | 3 颗子弹各自独立分裂（3+6=9 颗） |
| 弹幕地狱 + 弹幕风暴 | 3 颗子弹各自分裂并爆炸（3 大爆炸 + 6 小爆炸） |

## 4. 公式

### 子弹飞行
```
bullet.x += cos(direction) × bulletSpeed × deltaTime
bullet.y += sin(direction) × bulletSpeed × deltaTime

变量：
- direction: 子弹飞行方向（弧度）
- bulletSpeed: 260 px/s（默认）
- deltaTime: 帧间隔（秒）
```

### DPS 计算
```
dps = damage × bulletCount × (1000 / fireRateMs)

示例（默认值）：
- 10 × 1 × (1000 / 280) = 35.7 DPS

弹幕风暴：
- 10 × 3 × (1000 / 280) = 107.1 DPS（纯子弹伤害）

爆裂弹总伤害（单颗）：
- 子弹伤害 + 爆炸伤害 = 10 + 10×0.6 = 16（对单个敌人）
- 爆炸范围伤害 = 10×0.6 = 6（对范围内每个其他敌人）
```

### 多子弹角度分配
```
if bulletCount == 1:
    angles = [attackDirection]
else:
    for i = 0 to bulletCount-1:
        angles[i] = attackDirection + spreadAngle × (i / (bulletCount - 1) - 0.5)

示例（弹幕风暴：3 颗，spreadAngle = 30°）：
- i=0: direction + 30° × (0/2 - 0.5) = direction - 15°
- i=1: direction + 30° × (1/2 - 0.5) = direction + 0°
- i=2: direction + 30° × (2/2 - 0.5) = direction + 15°
```

### 爆炸范围判定
```
distance = sqrt((enemy.x - explosion.x)² + (enemy.y - explosion.y)²)
if distance <= explosionRadius:
    hit = true
```

### 分裂子弹角度
```
for i = 0 to splitCount-1:
    splitAngle = (2π / splitCount) × i

示例（splitCount = 2）：
- 分裂弹 0: 命中方向 + 0°（继续前进）
- 分裂弹 1: 命中方向 + 180°（反向）
```

## 5. 边缘情况

- **子弹命中同一帧多个敌人**：按距离排序，最近的先处理。穿透弹穿透最近的继续飞
- **穿透弹子弹已穿透过的人再碰到**：记录已命中敌人 ID，跳过重复命中
- **爆炸范围内击杀敌人**：爆炸伤害可触发 onKill，但爆炸不触发 onHit 触发器（只有子弹直接命中触发）
- **分裂子弹飞出屏幕**：正常回收，不产生额外效果
- **子弹命中帧敌人已死亡**：跳过该敌人，穿透弹继续飞行
- **弹幕风暴 + 爆裂弹 + 分裂弹三者共存**：3 颗子弹各自命中→先爆炸→再分裂（分裂弹不爆炸，除非弹幕地狱）
- **分裂弹 + 穿透弹**：分裂弹不继承穿透，穿透只对原始子弹生效
- **bulletCount × splitCount 数量过大**：弹幕风暴(3) + 分裂弹(2) = 最多 9 颗子弹，性能可控
- **子弹与子弹不碰撞**：只检测子弹与敌人的碰撞

## 6. 依赖

### 上游依赖
- **角色系统**：提供攻击参数（damage, bulletCount, piercing, fireRateMs 等）
- **移动与输入**：提供攻击方向和攻击触发信号
- **卡牌系统**：运行时修改攻击参数

### 下游依赖
- **敌人系统**：敌人接收伤害和状态效果
- **掉落物系统**：敌人死亡时触发掉落
- **HUD**：子弹视觉、爆炸特效、命中反馈

| bulletSpeed | 260 px/s | 150-400 | 子弹飞行速度 |
