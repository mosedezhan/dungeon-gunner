# 波次系统 GDD

## 1. 概述

控制敌人的生成节奏、数量和种类组合。每波次按固定间隔触发，随波次递增引入新敌人类型、增加敌人数量，并通过 HP 缩放提升整体难度。

## 2. 玩家幻想

玩家感受到明显的"一波接一波"的节奏感——每波之间有短暂喘息整理走位，波次开始时敌人从四面八方涌来。前期只有简单近战兵，后期多种敌人混合出现，逼迫玩家不断调整策略。

## 3. 详细规则

### 波次流程
- 游戏开始后立即进入第 1 波
- 当前波所有敌人被击杀后，进入波次间隔
- 波次间隔结束后自动开始下一波
- 波次无上限，持续到玩家死亡

### 波次间隔
- 间隔时长：3 秒
- 间隔期间屏幕中央显示"Wave X"提示
- 间隔期间玩家仍可移动

### 敌人生成

#### 每波敌人数量
```
baseCount = 5 + floor(waveNumber × 1.5)
```

#### 敌人种类权重
每种敌人有解锁波次，解锁前不出现。解锁后按权重随机选择：

| 类型 | 解锁波次 | 基础权重 |
|------|---------|---------|
| 普通近战 | 1 | 40 |
| 快速兵 | 3 | 25 |
| 爆破兵 | 4 | 15 |
| 射子弹 | 5 | 20 |
| 重装兵 | 7 | 10 |

权重随波次递减已解锁低级敌人的占比，让高级敌人逐渐成为主流：
```
actualWeight = baseWeight × (0.95 ^ (waveNumber - unlockWave))
```
最低权重不低于 5。

#### 生成位置
- 从屏幕边缘外随机位置生成
- 生成点距离屏幕边缘 50-150px
- 同一波次敌人分批生成，间隔 200-400ms，避免瞬间堆积

### HP 波次缩放
- 所有敌人 HP 按 `baseHp × (1.15 ^ (waveNumber - unlockWave))` 缩放
- 仅在该敌人解锁后的波次开始缩放

### Boss 波次（待定）
- 每 5 或 10 波出现 Boss（待设计）
- Boss 波次可能有特殊生成规则

## 4. 公式

### 每波敌人数量
```
baseCount = 5 + floor(waveNumber × 1.5)

示例：
Wave 1:  5 + 1  = 6
Wave 3:  5 + 4  = 9
Wave 5:  5 + 7  = 12
Wave 10: 5 + 15 = 20
```

### 敌人种类选择（加权随机）
```
对于每种已解锁敌人：
  actualWeight = max(5, baseWeight × (0.95 ^ (waveNumber - unlockWave)))
从 actualWeight 构建概率分布，随机选择 baseCount 次
```

### 生成位置
```
edge = random_choice([top, bottom, left, right])
if edge == top:    spawnPos = (random(screenWidth), -offset)
if edge == bottom: spawnPos = (random(screenWidth), screenHeight + offset)
if edge == left:   spawnPos = (-offset, random(screenHeight))
if edge == right:  spawnPos = (screenWidth + offset, random(screenHeight))
offset ∈ [50, 150]
```

### 分批生成
```
batchSize = ceil(baseCount / 3)
batchDelay = 300ms（固定）
每批生成 batchSize 个敌人，间隔 batchDelay
```

## 5. 边缘情况

- **波次内敌人全部出屏消失**：不计入击杀，但视为波次结束，触发下一波间隔
- **间隔期间敌人未清完**：不会出现，间隔仅在所有敌人死亡/消失后触发
- **高波次权重归零**：`max(5, ...)` 保证每种已解锁敌人至少有 5 的权重
- **同时有多种敌人生成**：每次独立抽取，可能同一波全是同一种
- **Boss 波次**：Boss 出现时暂停普通波次生成（待 Boss 系统设计后确认）

## 6. 依赖

### 上游依赖
- **敌人系统**：提供敌人类型、属性、AI 行为
- **角色系统**：需要玩家位置确定生成方向

### 下游依赖
- **经验与升级**：波次击杀产生经验掉落
- **精英敌人**：高波次可能生成精英版本（待确认）
- **HUD**：显示当前波次编号和波次提示

## 7. 可调参数

| 参数 | 默认值 | 安全范围 | 影响方面 |
|------|--------|---------|---------|
| 波次间隔 | 3s | 1-8s | 喘息节奏 |
| 基础敌人数 | 5 | 3-10 | 早期密度 |
| 每波递增系数 | 1.5 | 0.5-3.0 | 数量增长速度 |
| 权重衰减率 | 0.95 | 0.85-0.99 | 高级敌人替换速度 |
| 最低权重 | 5 | 1-10 | 低级敌人最低出现率 |
| 生成偏移范围 | 50-150px | 30-300px | 生成距离 |
| 分批延迟 | 300ms | 100-800ms | 生成节奏感 |
| HP 缩放乘数 | 1.15 | 1.05-1.30 | 长期难度曲线 |

## 8. 验收标准

- [ ] 第 1 波立即开始，生成 6 个普通近战兵
- [ ] 所有敌人死亡后 3 秒显示"Wave X"提示并开始下一波
- [ ] 第 3 波开始出现快速兵
- [ ] 第 4 波开始出现爆破兵
- [ ] 第 5 波开始出现射子弹兵
- [ ] 第 7 波开始出现重装兵
- [ ] 敌人数量随波次递增（Wave 10 约 20 个）
- [ ] 敌人从屏幕边缘外生成，不在屏幕内突然出现
- [ ] 同一波次敌人分批生成，非瞬间全部出现
- [ ] 高波次高级敌人占比逐渐增加
